  • equal_range(vec.begin(), vec.end(), k) 寻找 vec 里面值为 k 的区间
  • c++ 的内存结构分为静态分配内存和动态分配内存
    • 静态分配内存是编译和链接是确定的
    • 动态分配内存是加载，调入，执行时分配的
    • .text 为代码段，静态分配，read only，相同进程共用
    • .data 或者叫 GVAR，数据段之一，存放已经初始化的非零全局变量，静态分配
    • .bss 数据段之一，存放未初始化和零值全局变量，静态分配
    • text 和 data 段在可执行文件中，bss 不在，由系统完成初始化
    • 动态分配内存包括堆和栈
    • 栈从高往低生长，max_size 编译时确定，速度快，空间小，保存程序中的局部变量，程序自主维护。
    • 堆从低往高生长，max_size 与系统内存以及虚拟内存有关，速度慢，空间大，需要自主维护。
    • 每个线程有自己的栈，堆空间共用。
  • c++ 虚函数相关
    • 每个包含虚函数的类都有自己的虚表，如果基类有虚函数，子类也有自己的虚表
    • 虚表是一个指针数组，存放着虚函数的函数指针，虚表中不包含普通函数的函数指针。
    • 虚表是属于类的，一个类拥有一个虚表，当对象被创建时，对象拥有自己类的虚表指针。
    • 基类指针调用虚函数，首先寻找对象的虚表，然后根据虚表调用虚函数
  • c++ static 关键字
    • 加在全局变量前，为静态全局变量，全局变量按照是否初始化为分配在 .data 或者 .bss，总而言之会分配到数据段，静态的好处是无法被其他文件使用。
    • 加在局部变量前，为静态局部变量，只能作用于定义的函数或语句块，其他与静态全局变量相同。
    • 加在函数前，为静态函数，无法被其他文件使用
    • 加在类成员变量前，为静态成员变量，属于类而不是某个对象，好处是专属某类，不与全局变量重名，可以信息隐藏。
    • 加在类成员函数前，为静态成员函数，属于类而不是某个对象，只能访问静态成员函数和静态成员变量。
