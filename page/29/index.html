2019 08 15 — 工作 今天的工作紧接昨天，昨天网络已经可以正常跑起来了，但是没有量化去跑，之前写的一些算子，都只测试了 forward 的代码，没有测试 function，module，backward 的代码，今天跑了量化发现，bug 非常之多。于是，整天都在 debug，而算法层面没有什么问题，所以 debug 的过程无非是一些手滑写错的代码，接口对不上等小问题。 具体来说的话，之前没有意识到的一个问题是，backward 要求的返回值也必须是 variable 的，不然会报接口错误，意识到这一点后，其实当时很多算子都没有考虑到，所以把这次用到的都改了一遍。另一个问题是，conv 算子没有考虑 bias 为 none 的情况，因为 kernel 里面默认 bias 是存在的，所以在 bias 为 none 的时候，把它直接赋值为0解决了这个问题。 经过不懈的努力，终于没有 bug 了。我很兴奋地开始跑我的网络，毕竟里面用的所有的算子比如 conv2d, batchnorm2d, maxpooling, upsample 等都替换成了我自己的写的。我发现速度不是很理想，大概要慢8倍左右，这个数字我是能预料到的，当 batch_size 更大的时候，会变得更慢，我知道问题所在，却也解决不了，我 mentor 说已经很快了，我毕竟不是搞 hpc 的，cuda 优化我确实不会。显存方面倒没有很大问题，就比 pytorch 用的算子多了 200MB 的显存占用。不知道以后有没有机会解决这个效率的问题。 代码跑通了之后，下个任务自然就是在集群上可以跑，要用 docker 打包成 image。写 dockerfile 文件并没有花费很长时间，我发现有个问题，当你 build dockerfile 文件的时候，它会分析当前文件夹的上下文，我的数据集都在当前文件夹下，虽然没有在 dockerfile 中 ADD 到 image 之中，但还是会分析到它们，会很浪费时间。我做了一些小的实验发现，把数据集拿掉就会快很多。于是，我把数据集统一放在了文件夹外，然后在文件夹内做了一个软链接解决了这个问题。 目前 dockerfile 已经写好了，应该可以在集群上跑，但是 oss 的问题一直没有得到解决，我没办法把数据集传到 cephfs 上面，只能把一些 toy dataset 打包进 image，其实没有什么实际意义。…
