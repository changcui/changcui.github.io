寄生虫

这个片子拿下了韩国第一座金棕榈，总体质量还是可以的，毕竟有宋康昊坐镇，印象比较深的镜头是女儿坐在马桶上抽烟，非常打动人。美中不足的是全片显得太刻意了，其实导演不必表达的如此刻意，相信大部分观众是能 get 到点的。

献给阿尔吉侬的花束

美国作家的一个科幻小说，讲述了一个智力障碍的人变成天才之后的故事，我还没读完，简介里说有一幕是变聪明了之后也想要戏弄一个同样有智力障碍的人，这一幕与寄生虫不谋而合。

小说的中文翻译很有意思，因为全篇都是主角的自述，主角有智力障碍时，写的日记有很多错别字，我差点想换一个版本，直到看了别人的见解才明白。

大学老乡群

在一个老乡群里，有学弟学妹，也有学长学姐之类的，我是没体会到老乡群给我的任何帮助，我尝试着加同一个公司的老乡也被拒了两次，一些学长学姐无非就是发一些消息，让学弟学妹带他们的朋友进校园。我向来不喜欢麻烦别人，我在群里年龄也算不小，所以应该也麻烦不了我，但我觉得这些学长学姐的做法并不妥。我与一个一起考来的同学讨论了一下，他立马就退群了，说早就不爽了，我没他这么果决，没选择退群。

香港事件

今天论坛里有关于香港事件内因的讨论，说到了很多我之前没有考虑过的点，我印象比较深的是郑永年教授的这篇文章，早在2015年就已经指出了问题所在，不算是马后炮吧。

香港虽然回归了，但是没有万事大吉，去殖民化没有得到很好的进行。虽然说是一国两制，大陆这边更强调一国，而香港那边却强调两制，国家的感觉十分淡漠。尤其是新一代的年轻人，更缺乏国家的概念。

政治体制也是原因之一，英治下的香港是没有政治的，政治属于英国，香港只有行政权，而回归之后，香港拥有了政治的话语权，却也无处表达，其最高行政长官不是从本身的政治体制得出的，特首没有党派的支撑，执行行政权势必会与议员之间势必会形成矛盾。

香港是资本主义社会，资本家拥有的话语权远远高于普通民众，而且地方发展的利益绝大多数掌握在少数利益集团手中，民众获得的利益太少了，当经济出现危机的时候，民众无法将此愤怒发泄到资本家身上，只能发泄到政府上面，进而对大陆形成敌对。

顺便查了一下郑永年教授的思想，发现很有意思，他认为西方所谓的民主法治应该是法治民主，这里的民主不仅仅指的是普选，在一个没有发展起来的国家做普选是没有意义的，民众太容易被小利小惠煽动行使自己宝贵的政治权利，应该是有经济和法治支撑的民主。他很推崇新加坡的制度，认为中国在将来也能走出一条属于自己的道路。

docker

今天的工作很少，玩着玩着就写完了，其实就是把之前写的算子和一个 aligner 的网络用 docker 封成一个 image，然后在 k8s 上跑起来就可以了，因为网络很 toy 其实就是走一遍流程，等下周有了更成熟的方案，就可以直接跑了。 mentor 让我等运维组的进度，顺便建议我休息一下，真的是官方指定摸鱼。

使用 docker 封装的时候其实遇到的都是一些环境的问题，基本上通过 google 都可以解决，这里总结一下：

  • pytorch 的 cpp_extension 在 build 的时候会有 ABI 问题，我查了一下，ABI 类似于 API，是代码转二进制时的接口，不同的编译器会有不同的接口，应该类似于 nvcc 在编译的时候要指定 sm_35 or sm_40 等吧。关于这个问题，解决办法是切换 gcc 的版本，我先把它降级到 4.9，不过发现和 cuda 的 driver 不兼容，当我把 cuda 的 driver 重新装好之后，它又默认帮我装上了 5.4，后来发现系统是可以同时兼容多个版本的 gcc 的，只需要制定一下哪个版本优先就可以了。

    sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-4.9 100

  • pytorch 的 cpp_extension 在 JIT 执行的时候，会调os.environ['CXX']的环境变量来判断编译器的版本，但是它代码里写的似乎有点问题，如果没有手动指定环境变量的话，会返回一个 c++ 的值，但 c++ 不是一个编译器，这样就会出错。有三种解决方案，第一种是修改 pytorch 的源码，但是对于 docker image 而言不太现实，第二种是设置环境变量，要改程序的代码，不太方面，最后使用的是第三种方案，在 dockerfile 里面加一行 RUN alias c++='g++' 让 c++ 当 g++ 的别名，就解决了这个问题。

  • 然后运行的时候发现会出现 cuda runtime error 的问题，这个问题在于 cuda 的环境变量没有设置好，我猜想应该是在装 cuda driver 的时候默认设置好了，但是在 container 环境中没有这个步骤，只需要再加一行 ENV CUDA_HOME=/usr/local/cuda 就可以成功运行了。

  • 然后发现 dockerfile 里面如果想要 add 文件夹的话，不能写成 ADD foo /var 这样写是把 foo 里面的东西加入 var，需要写成 ADD foo /var/foo

华为鸿蒙

快下班的时候看到了这个消息，华为推出了自己的鸿蒙系统，采用了微内核的架构，没有 root，我心想大概骗一骗不懂操作系统的人还行，没有 root 的操作系统是不可能的吧，其实就是意味着不开放 root 呗，何必去炒作 root 这个概念，说没有 root 来强调自身的安全性呢。

另外，有一张现在和未来的图片，现在的架构里面同时有鸿蒙内核和 linux 内核，意味着现阶段没有 linux 内核还是不行的，想要像吹得这么厉害，一个内核所有硬件都可以跑，看起来还需要很长时间。

现在手机的宣传有两种感觉，一个是制造概念，明明不是什么高级的东西，非要抛出一个看似高级的概念来误导群众，另一个是华而不实，总是推出一些没什么用的功能，麻烦你把软件做好，把屏幕做好再推出这些功能吧。

不过，写一个操作系统的内核的工作量是非常大的，还是很佩服华为有这样的底气与决心，还将代码开源，是要打造自己的生态吧，希望华为可以成功。
