Arch 踩坑日记

Posted at — Mar 13, 2020

2020 03 12 update

完成 iso 下载，开始准备安装，参考教程

https://wiki.archlinux.org/index.php/Installation_guide_(简体中文)

https://github.com/mytbk/Linux_Notes/tree/master/install

这次先用 virtualbox 做实验把，下次争取能装双系统！

出师不利啊，刚开始装 virtualbox 连新建虚拟机都失败了，试着更新一下把，家里的网还巨慢，100 KB/s，大大降低了我折腾的效率，这能借助手机开热点外加翻墙了。。

果然可以初始化了。

第一次安装失败，按照教程一步步走下来了，但是 reboot 的时候不太对劲，我觉得是 UEFI 或者 BIOS 的问题。

如果你想在 VirtualBox 里以 EFI 模式安装 Arch Linux，这需要在虚拟机的设置窗口里，先从左侧选择*System*，再从右侧选择*Motherboard*标签页，最后勾选*Enable EFI (special OSes only)*选项。从安装镜像的启动菜单里选好 Arch Linux 的内核之后，这时启动过程要卡顿一两分钟，之后就能正常进入系统。稍安勿躁。

果然，我没开启这个设置，只能再来一遍啦。

第二次安装失败，这次可以 UEFI 启动，但是只有一个启动选项是 firmware interface，不知道原因在哪。

按照 wiki 里面的说法，要手动选择 UEFI，可是我手动选择了也不对，感觉是有地方搞错了，再来一遍把。

第三次失败，和第二次的情况一样，暂时不知道原因，查了一下似乎是 gummiboot 的问题，把 bootloader 切换成 grub 再试一次把！

第四次，成功进去了，不过是自己找 EFI 文件引导的，后面还得调整一下，怎么能自动引导，然后就是不能上网，好像是没有安装 networkmanager

找到原因了，是因为我没有移除 archlinux.iso 作为 attachment，然后它找 EFI 文件的路径发生了变化，没有从 /boot/EFI/BOOT/ 开始找，把 iso 文件移除之后，现在能正常启动啦！

不能上网似乎是少装了 linux-firmware，然后我搞不定 virtualbox 的 shared folder，也没办法用 ssh 来传文件，明天再装一遍把。

2020 03 13 update

今天试着重装一遍总结一下步骤。

安装前准备，virtualbox，archlinux.iso 文件，然后新建虚拟机，去 setting -> system -> motherboard 启动 UEFI，调整 display 为二倍显示，不然看起来会很毁眼。。

  1. 启动安装，修改 /etc/pacman.d/mirrors.txt，最好改成大陆的源，比如 Server = https://mirrors.ustc.edu.cn/archlinux/$repo/os/$*arch*.

  2. 然后gdisk来硬盘分区，o初始化分区，n创建分区，至少创建一个 EFI 的引导分区，和一个 linux file system 的装软件的分区，也可以创建 swap 分区

  3. 设立分区文件系统，使用mkfs.xx来创建文件系统，EFI 使用 FAT32 文件系统

  4. 挂载，先挂载根目录，再依次挂载mount $分区名 $挂载点

  5. 生成挂载信息保存，开机自动挂载genfstab -U /mnt >> /mnt/etc/fstab

  6. 安装内核，固件等，pacmanstrap /mnt base linux linux-firmware networkmanager base-devel

  7. chroot，arch-chroot /mnt

  8. 安装编辑器，dhcp 服务等，pacman -S vim dhcpcd

  9. 修改 root 的密码，passwd

  10. 安装 bootloader，pacman -S grub efibootmgr

  11. 配置 bootloader，首先初始化，grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB --removable ，最后的整个 removable 的参数是 virtualbox 里面需要使用的，然后生成配置，grub-mkconfig -o /boot/grub/grub.cfg

  12. 退出 chroot 模式，reboot

  13. unattach iso 文件，就可以成功进入了

关于网络的问题，需要安装 dhcpcd，然后 systemctl enable dhcpcd@enp0s3.service，就是给你的网卡启用 dhcp 服务的意思，然后就可以正常使用了。
