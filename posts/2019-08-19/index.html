思念是一种病

工作

今天的工作基本上就是划水，整天没有和任何一个人说过任何一句话，听了两个评书和四个电台，确实是非常无聊了，运维组那边依旧没有进展，不知道什么时 候才会有活干，仿佛也提不起来兴趣去看 paper，看代码。

Termux

在划水的时候，我看了少数派的一个文章，讲 termux 的，termux 是安卓机上的一个 termial，之前我下载了，一直不知道怎么用，看了这个文章之后，觉 得还是非常有意思的，于是我一下午基本上都在折腾这个东西。

  1. 首先，我先改了一个软件源，改成了清华源的镜像，下载速度果断快了很多，但我用的是4G，不是公司的 WIFI。
  2. 然后，我配置了一下 zsh，装了 oh my zsh 的 termux 版本，换了 solarized 的 theme，看起来舒服多了，虽然字很小吧。
  3. 然后，配置了 vim，用的是 github 上我用的配置，但是 snippet 好像用不了，我也不打算用它来写代码，直接给卸载啦。
  4. 然后，装了 python 和 clang，因为用 python，很多别的软件都可以完美运行了，比如 you-get，可以用来下载视频啦，clang 可以对付一些需要 c 的环境，比如 python 的 numpy。
  5. 然后，装了 aria2c，一直非常好用的下载器，可以用来下载百度云盘，虽然我不太用得到，毕竟用手机下载大文件的场合屈指可数，平时用 IDM+ 也够用，多线程下载甚至更快。
  6. 最后，其实我最想干的一件事就是把 v2ray 放在后台，因为现在用 bifrostv 总觉得不好用，没法使用电脑上那种配置。我本来想得很简单，把 v2ray 运行到后台，使用 1080 端口来接收 socks5 的包，然后让 app 都走 sock5 去 1080 端口。但是，具体做起来没有我想象中的简单。以 instagram 为例，instagram 的 client 没有 proxy 的设置，所以虽然我架好 v2ray listern to 1080 port，但是我不知道怎么讲 instagram 的流量对应到 1080 端口，这个问题就解决不了。但是一些有 proxy 设置的 client，比如 fire fox 或者 telegram，这种方法确实是可以使用的。在网上查一下，如果想让所有的流量都走相应的端口，需要设置 vpn，但是这个对我来说太麻烦了，不是一两个命令能解决的问题。我看我下载的一些 client，他们采用的也是这样的方式，看了一下他们的代码，应该是用了 android 提供的 vpn 接口，就是不知道怎么讲 vpn 的协议与 socks5 对应上的，可能以后还要再研究研究。
