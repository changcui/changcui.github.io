生日

今天是我23岁的生日，感觉上还不算老，但是和朋友聊天时变得喜欢怀念过去，饮食还是一样的肚子却慢慢大了起来，虽然我的头发还没有开始秃，但时间好像真的在我身上流逝，不知不觉地触及皮肤，触及躯干，触及灵魂。

我是能感到思想的改变的，我的价值观也在不停地变，将来还会不会变呢，是不是形成固定的价值观，人就是变成熟了，也变固执了？

我是能感到自己的成长的，至少这一个暑假，这一份实习，我觉得自己提高了很多，最明显地就是面对困难的态度，以前总想着太麻烦，不想动手，现在就是要把它做出来，get your hand dirty，就是这个含义吧，一步一步地走下去，路总会越来越简单。

我是能感到自己的喜好的，很长时间我是不知道自己喜欢什么的，我认为很多事情都很 valuable，但是实际上并没有很多时间去做，优先去做的事情大概就是喜好吧，我不喜欢无聊的人际交往，我只希望和小部分人一起玩，我喜欢科技产品，喜欢柯基，喜欢单机游戏，喜欢女票。

如果能许个心愿的话，希望一切都好吧。

工作

今天一早就起来去工作了，喝了久违的皮蛋粥，感觉比蔬菜粥好喝一些。并没有刷很长时间的社交媒体，就开始干活了。mentor 说我一天干一个小的任务还真不错，昨天搞完了第一个 subtask，今天从第二个入手。

先去找 mentor 讨论了一下困境，然后我就开始努力地 coding，很快就写完了，因为就是把之前做的两个东西融合到一起，但是总是差一点点，我本来以为无所谓，但是 mentor 说应该是完全一样的，检查了好久发现是因为 python 和 c 在 float 的处理上是不一样的，python 的 float 默认的是64位小数，对应的是 c 里面的 double，在 python 和 c 交互的时候，会自动作截断，导致比原本的整数小一些，然后转 int 的时候会取 floor，产生 1 的差异，做了卷积运算后，差距就变大了。

察觉到这一点 bug 后，我讲输入加了个 0.5 再转 int，成功解决了这个问题。顺便解决了第三个 subtask，其实很简单，就是把 scheduler 的状态 save 一下就可以了。

下午开会之前，我仔细地重新捋了一遍算法的流程，找 mentor 讨论了一下关键点，讨论的过程中想出了一个更好地解决方案，mentor 说这个方案非常不错，于是我往 subtask 里面又多加了几条。

下午开会我说了不少话，基本上占了1/3的会议篇幅吧，讨论了一下我的困惑以及需要的帮助，总的来说这次会议还是比较顺利的，进行地很快，很有效率，我喜欢这个节奏。

晚餐

晚餐是常规的生日标配四人组，这次有点不一样，不是在家里吃的，是在外边的饭店吃的。我觉得吃的还可以，但是应该没有下一次了，因为没有太突出的地方， 环境也一般般，特别是馕，像压缩饼干一样，配合酸奶效果奇佳。若干年后，我想不起23岁生日的内容，但是应该还是能记起吃过的馕吧。

去向

和女票讨论了一下去向的问题，马上就要毕业了，将来应该去哪里呢，留在帝都的话，没有户口，但是工作的发展不错，朋友也都在，去上海有户口，钱挣够了就可以买房，也许结婚能早一点？工作的发展应该还可以，但不如北京稳妥吧，朋友自然是没有了，去小城市生活压力应该不到，但是职业的发展会遇到瓶颈吧，一系列的问题摆在面前，仿佛没有一个最优的决策。

其实当然没有最优决策，我是觉得无所谓的，我去哪都可以，北京或者上海，我找个工作应该都没有什么问题，女票仿佛容易受到外界的干扰，一些小的信息就可能让她的决定改变，我想了想，大概是因为我一直都没有很坚定的选项，当然也没有给她足够的支持。

长久以来，对于去向的问题，女票似乎总是主导的一方，但是又常常改变，我不知道面临这种情况我应该怎么处理，也许我应该确定自己想要去的地方，才能更好地帮她做决定？
